Arquitetura de Segurança

Network Access Control (NAC)

Estratégias para Redes Zero Trust

Da visibilidade total ao controle de acesso automatizado: como proteger redes modernas onde o perímetro desapareceu.

Zero Trust 802.1X Compliance

1. O Fim do Perímetro

O modelo tradicional de "Castelo e Fosso" tornou-se obsoleto. Com BYOD, IoT e trabalho remoto, a identidade é o novo firewall.

Desafios Modernos

BYOD & IoT
Cloud & SaaS
Policy: "Never Trust, Always Verify"
Scope: User, Device, Location, Health

2. Visibilidade Total

"Você não pode proteger o que não vê". O NAC utiliza múltiplas técnicas de Device Profiling para criar um inventário em tempo real.

DHCP
Análise de fingerprints (Option 55) para identificar S.O.
NMAP
Active Scanning para descobrir portas e serviços.
SNMP
Leitura de tabelas MAC e ARP dos switches.

3. Enforcement (802.1X)

O padrão ouro para controle de acesso. Bloqueio na Camada 2 (Porta do Switch) antes mesmo da atribuição de IP.

switch(config)#
! Configuração Típica 802.1X
aaa new-model
aaa authentication dot1x default group radius
interface GigabitEthernet1/0/1
authentication port-control auto
dot1x pae authenticator
EAP-TLS Certificados Digitais (Máxima Segurança)
MAB MAC Bypass para impressoras/IoT (Legado)

4. Resposta Automatizada

Integração com EDR e SIEM para contenção imediata via Radius CoA (Change of Authorization).

1. Ameaça Detectada

EDR detecta Malware no Host X.

2. Alerta ao NAC

Webhook/API informa o Policy Node.

3. Quarentena (CoA)

NAC envia comando ao Switch: "Mover porta para VLAN Quarentena".

Conclusão Estratégica

A segurança moderna migrou do dispositivo para a identidade. O NAC não é apenas uma ferramenta de bloqueio, mas o orquestrador central que une Infraestrutura, Conformidade e Resposta a Incidentes.

Key Takeaways

  • Segmentação dinâmica é vital.
  • Integração (API) supera funcionalidades isoladas.
  • Fail-Open vs Fail-Close deve ser uma decisão de negócio.
