Container Security & Hardening

Securitização de infraestrutura baseada em Docker. Foco em imagens seguras, isolamento de recursos e controle de acesso mTLS.

1. Imagem Base Hardened

Construção a partir de S.O. limpo, evitando imagens "black-box" e aplicando o princípio do menor privilégio.

FROM ubuntu:latest
ENV DEBIAN_FRONTEND=noninteractive
RUN apt-get update && apt-get install -y nginx

2. Mutual TLS (mTLS)

Proxy Reverso configurado para exigir certificados válidos de ambos os lados, garantindo que apenas clientes autorizados acessem a API.

# nginx.conf
ssl_client_certificate /etc/nginx/ssl/ca.crt;
ssl_verify_client on;
